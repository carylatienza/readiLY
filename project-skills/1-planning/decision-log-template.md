# Decision Log

> **How to use:** One table row (or entry) per significant decision — technical or product. Add entries the moment a decision is made, while the reasoning is fresh. Never delete entries; if a decision is reversed, add a new entry and link back. This file answers "why is it like this?" six months from now.
>
> **What counts as significant:** anything expensive to reverse, anything two people disagreed about, anything a new teammate would question.
